# 微信公众号文章获取工具

一键获取任意微信公众号的全部文章 URL 及纯文本正文。
本工具提供两项核心功能：
1. **单篇文章读取**：提供任意单篇微信公开文章的链接，即可稳定解析出标题与正文纯文本。
2. **公众号全量提取**：通过微信公众平台配合本地登录凭证，可一键提取目标公众号的全部（或指定时间段）历史文章及纯文本正文内容。

## 快速开始

### 1. 准备环境

在终端输入以下命令安装必要的依赖：
```bash
pip install -r requirements.txt
playwright install chromium
```
> **🤖 如果使用 AI Agent（示例）：**
> 您可以直接对智能助手发指令：“帮我检查并安装这个项目的依赖（requirements.txt 和 playwright 内核）”

### 2. 指定目标并运行

在终端运行主程序并指定目标公众号名称：
```bash
# 进入交互菜单：继续上次、输入公众号新建任务、只分析历史任务
python crawler.py

# 菜单中选“2”，直接输入公众号名称新建任务，不用改 config.json

# 直接执行，不进菜单
python crawler.py -a by --no-menu

# 用 config.json 里的短别名抓取，少打长名称
python crawler.py -a by

# config.json 只有一个目标时，也可以直接运行
python crawler.py

# 默认自动续跑：复用最近一次 output/公众号名_时间戳/ 里的进度
python crawler.py -a by

# 如果确实想重新开一个全新任务目录
python crawler.py -a by --new-run

# 仅获取最新的 10 篇文章
python crawler.py --nickname "数字生命卡兹克" --max 10

# 临时换公众号抓取，不修改 config.json
python crawler.py --nickname "公众号名称"

# 获取从指定日期以来的所有文章
python crawler.py --nickname "数字生命卡兹克" --since 2026-03-01

# 日期 + 数量组合使用
python crawler.py --nickname "数字生命卡兹克" --since 2026-03-01 --max 50

# 高并发 + 多代理抓正文（正文阶段默认 16 线程）
python crawler.py --nickname "数字生命卡兹克" --max 50 --workers 24 --proxy http://127.0.0.1:7890 --proxy socks5h://127.0.0.1:1080

# 从代理池文件读取，每行一个代理，也支持逗号/分号分隔
python crawler.py --nickname "数字生命卡兹克" --max 50 --workers 32 --proxy-file proxies.txt
```
> **🤖 如果使用 AI Agent（示例）：**
> 您可以直接给出自然语言需求：“请帮我抓取公众号《数字生命卡兹克》最新的 10 篇文章。” 或者 “帮我获取《某某科技》2026-03-01 以来的所有文章全文。”

### 并发线程与代理配置

正文抓取阶段默认开启高并发，`config.json` 里的默认值为 `content_workers: 16`、`content_delay_seconds: 0`。文章列表阶段仍按顺序翻页并保留 `delay_seconds`，避免微信公众平台接口频控。

可用方式：
- `--workers 24` 或 `--content-workers 24`：设置正文抓取线程数。
- `--proxy http://host:port`：添加一个代理，可重复传入。
- `--proxy "http://a:8080,socks5h://b:1080"`：一次传入多个代理。
- `--proxy-file proxies.txt`：从文件读取代理，每行一个，`#` 开头为注释。
- 支持 `http://`、`https://`、`socks5://`、`socks5h://`；未写协议时默认按 `http://` 处理。

`config.json` 也可以直接配置：
```json
{
  "crawl_settings": {
    "content_workers": 32,
    "content_delay_seconds": 0,
    "content_min_workers": 4,
    "content_proxies": [
      "http://127.0.0.1:7890",
      "socks5h://127.0.0.1:1080"
    ],
    "content_proxy_file": "proxies.txt"
  }
}
```

代理池会按文章序号轮询分配；如果失败率过高，默认会自动降并发，避免整批请求连续失败。

### 短别名

`config.json` 的目标支持短别名：
```json
{
  "targets": [
    {
      "nickname": "宝医社区健康管理",
      "alias": "by"
    }
  ]
}
```

之后可以直接运行：
```bash
python crawler.py -a by
```

输出目录默认使用公众号名字，例如 `output/宝医社区健康管理_20260522_120000/`；`alias` 只负责让命令更短。

### 断点续跑

默认开启断点续跑。再次运行同一个别名时，会自动找最近的 `output/别名_时间戳/`：
- `article_list.json` 已完成：跳过文章列表阶段。
- `article_full.json` 已完成：跳过正文抓取阶段。
- 只有部分正文：只补缺失正文，不重新抓已有正文。
- `分析报告.md`、`非微信网络资产.txt`、`图片资源/本地图片清单.txt` 都存在：跳过分析阶段。
- 旧版 `output/公众号_时间戳.json` / `output/公众号_full_时间戳.json` 会自动迁移成新的续跑目录。

强制重新开始时使用：
```bash
python crawler.py -a by --new-run
```

如果已保存的 `credentials.json` 过期，程序会自动打开浏览器重新扫码获取 Cookie/Token，然后重试当前任务；也可以手动加 `--relogin` 强制重新扫码。

### 3. 获取登录凭证 (核心步骤)

微信公众号文章对全量获取有严格的扫码与 IP 安全限制。**您必须根据当前的运行环境采取不同的策略**：

#### 场景 A：在本地个人电脑（Windows/Mac）上运行

直接运行上述终端命令。由于未提供凭证，程序会自动在您的电脑上弹出一个可视化的 Chrome 浏览器网页，您只需要**拿出手机微信扫一扫屏幕上的二维码**，程序会在登录成功后自动接管剩下的目录翻页和正文下载工作。

> **🤖 如果使用 AI Agent（示例）：**
> 记得向助手说明环境：“我是在本地运行，请帮我启动全自动浏览器扫码获取程序。”

#### 场景 B：在云服务器（无桌面的 Linux / 云机房节点）上运行

微信对云厂商 IP 节点及无头浏览器风控极其严格，切勿在云端直接执行登录！您必须**手动在本地电脑提取凭证后再交给服务器运行**：

**手动操作步骤：**
1. 用您自己本地电脑的浏览器登录 [微信公众平台](https://mp.weixin.qq.com)。
2. 按 `F12` 打开开发者工具，切到 **Network (网络)** 标签页。
3. 刷新页面，随便点击左侧一个以 `cgi-bin/` 开头的网络请求。
4. 从 URL 地址里复制出 `token=数字` 里的数字。
5. 往下看面板，在 **Request Headers (请求标头)** 里找到 `Cookie: ` 字段，把后面一整串极长字符完整复制下来。

最后在云服务器终端运行带参数的命令（将参数替换为您提取的实际值）：
```bash
python crawler.py --nickname "目标公众号" --credentials '{"cookie":"填写刚才你的长串Cookie","token":"填写刚才你的Token"}'
```

> **🤖 如果使用 AI Agent（示例）：**
> 您不需要亲自手写 JSON，直接把这两个纯文本扔给 Agent 组装：“这是我的 Cookie：xxxxx，这是我的 Token：123456。凭证给你了，我在云服务器，直接帮我跑静默抓取吧！” 

### 4. 自动产生全量结果

程序已实现全自动连贯执行：
1. 提取文章基础目录；
2. 自动进入第二阶段下载所有文章纯文本详情（Content）；
3. 自动进入第三阶段生成资产/图片/网络链接分析报告。

每次运行都会在 `output/公众号名_时间戳/` 下生成一个完整任务目录，常见文件包括：
- `article_list.json`：文章列表 checkpoint。
- `article_full.json`：带正文的完整结果。
- `分析报告.md`：正文、图片、网络资产汇总报告。
- `网络资产.txt`：去重后的网络资源 URL。
- `非微信网络资产.txt`：去掉 `qq.com`、`qpic.cn`、`weixin.qq.com` 等微信/Tencent 域名后的网络资源 URL。
- `图片资源/`：图片资源清单、本地图片、疑似二维码/小程序码图片清单。
- `图片资源/本地图片/`：下载到本地的图片文件，便于直接查看二维码、海报等内容。
- `图片资源/疑似二维码小程序码图片.txt`：基于 URL/标题/上下文关键词筛出的疑似二维码或小程序码图片，便于人工重点检查。
- `图片资源/标准二维码解码结果.txt`：如果安装了 `opencv-python`，会尝试解码标准二维码；微信小程序码通常需人工查看或用微信识别。
- `资产明细/`：按类型拆分的资源清单。

如果只想爬正文、不生成分析报告，可以加：
```bash
python crawler.py --nickname "目标公众号" --skip-analysis
```

## ⚠️ 核心使用受限说明与注意事项

为了能长久稳定使用本脚本，请务必仔细阅读以下受限说明：

1. **凭证不具备持久性（几小时内即失效）**：
   提取出的 `Cookie` 和 `Token` **绝不是永久的**！微信的安全机制决定了扫码登录态寿命极短，通常在 **几小时甚至几十分钟（如果你变更了IP）** 就会过期。一旦程序报错需要重新提取，说明凭证已自动失效，你必须马上跑去公众号后台重新扫或者重新提取，**世界上没有永久绕过扫码拿数据的方法**。
2. **云服务器自动化受限（封号风险）**：
   **请勿**在云服务器（如阿里云、腾讯云机房节点）等非常规地理位置 IP 上尝试启动浏览器的全自动扫码登录，微信对机房 IP 风控极其严格，一旦被风控，该公众平台账号可能直接被封禁网页端登录权限。请遵守【场景 B】的规定拿取本地凭证参数投喂！
3. **抓不到点赞/阅读量等互动数据**：
   受限于只能获取网页文章列表页设计，本程序**只能完美获取“所有文章元数据列表”和“文章内部的纯文本（Content）”**。至于一篇文章的阅读数、点赞数、评论数等，因保护极严只能通过手机客户端专属流量获取，纯外部浏览器抓取程序是做不到的。
4. **频率限制**：
   单个账号请求抓取间隔默认 **3 秒**，若获取超过几千篇的老号，高频狂刷容易导致 IP 被微信官方拦截 24 小时。遇大量任务请在 `config.json` 里适当调高 `delay_seconds`。
5. 本仓库仅用于个人学习研究，请尊重内容版权与数据所有权。
